| [Proximal Policy Optimization (PPO): The Key to LLM Alignment](https://cameronrwolfe.substack.com/p/proximal-policy-optimization-ppo) | Reinforcement learning has evolved from methods such as Deep Q-Learning and Vanilla Policy Gradient. Vanilla Policy Gradient suffers from high variance and low sample efficiency because policy updates are estimated from sampled trajectories, making learning unstable.

To improve stability, algorithms such as TRPO and PPO were introduced. TRPO maximizes a surrogate objective while constraining the KL divergence between the old and new policies, ensuring that policy updates do not change too drastically. KL divergence measures the difference between two probability distributions and helps maintain stable learning.

PPO simplifies TRPO by replacing the hard KL constraint with a clipped objective function, making the algorithm easier to implement and more computationally efficient while still preventing excessively large policy updates.

In RLHF, humans rank outputs generated by a language model. These rankings are used to train a reward model that predicts human preferences. PPO is then used to fine-tune the language model to maximize the reward while often including a KL divergence penalty to prevent the model from drifting too far from the original pretrained policy. |
| [Basics of Reinforcement Learning for LLMs](https://cameronrwolfe.substack.com/p/basics-of-reinforcement-learning?utm_source=publication-search) | Reinforcement learning is a learning approach in which an agent interacts with an environment and learns how to make decisions by receiving rewards. The goal is not only to maximize the immediate reward from one action, but to maximize the expected cumulative reward over time, which is called the return. This process is often modeled as a Markov Decision Process, where the agent observes a state, chooses an action according to a policy, receives a reward, and moves to the next state. Q-learning is a reinforcement learning algorithm that learns the value of each state-action pair using a Q-table. However, when the number of states and actions becomes very large, storing all values in a table becomes inefficient. Deep Q-learning solves this problem by using a neural network to approximate the Q-values instead of storing them directly in a table. |
| [Decision Transformer](https://arxiv.org/abs/2106.01345) | Casts offline RL as sequence modeling. A GPT-style transformer conditioned on return-to-go, state, and actions is trained with supervised learning — no Bellman backups needed. |
